# 저장소 durability: unlimitedStorage + 수동 전체 백업

local-first 메모리 제품의 급소는 데이터 분실이다. v1은 두 겹으로 막는다: (1) 크롬 익스텐션의 `unlimitedStorage` 권한 + `navigator.storage.persist()`로 용량 압박에 의한 자동 삭제(eviction)를 막고, (2) "전체 메모리 백업/복구 파일"로 익스텐션 삭제·재설치·기기 분실에 대비한다. 백업 파일은 익스텐션 바깥(사용자가 고른 위치)에 있어야 삭제·재설치에도 살아남는다.

## Considered Options

`unlimitedStorage`는 자동 eviction은 막지만 익스텐션 삭제·기기 고장·다기기 분산은 못 막는다. 이는 "한 기기에만 있다"는 단일 실패점이라 기기 밖 한 벌이 필요하다. 폴더 자동 백업(File System Access 재사용)도 후보였으나 v1에선 수동 백업까지만 넣고 자동 백업은 다음으로 미룬다. 기기 고장까지 막는 완전한 답은 유료 클라우드 sync다(ADR 0011).

## Consequences

- 백업에는 원문 텍스트+메타+하이라이트(대체 불가 자산)를 담는다. 벡터는 복구 시 재임베딩으로 재생성 가능하므로 기본 제외해 파일을 가볍게 한다(옵션으로 포함하면 빠른 복구).
- 복구는 백업 파일을 읽어 DB를 재구성한다.
- 자동 폴더 백업(FS Access 재사용)과 유료 클라우드 sync는 durability를 더 높이는 후속 단계다.
- 저장 규모는 대량(수만 페이지)에서 벡터가 용량의 대부분(약 1GB대)을 차지한다. 압박이 생기면 int8 양자화로 벡터를 4배 줄이는 것이 레버지만 v1엔 불필요하다(ADR 0002).
