# 캡처 게이트: 프라이버시는 빡세게, 참여는 관대하게

게이트가 틀리는 두 방향의 비용은 극단적으로 비대칭이다. False positive(쓰레기 캡처)는 저장·잡음 약간이고 랭킹이 묻거나 삭제로 복구된다. False negative(필요한 걸 안 담음)는 답이 영영 사라지고 검색 실패 전까진 보이지도 않는 복구 불가 손해다. 이 제품의 존재 이유가 "옛날 그것을 찾기"이므로 false negative가 더 치명적이다. 따라서 Hard gate(프라이버시: denylist·시크릿·금융/헬스)는 공격적으로, Soft gate(참여: dwell·thin page)는 관대하게 둔다. 게이트가 버렸을 페이지를 위한 Manual save를 v1에 둔다.

## Considered Options

원래 1-pager는 engagement-gated capture를 "가장 레버리지 큰 결정"으로 두고, 캡처 건강도 지표를 "게이트 통과율(이상적으로 절반↑ 걸러짐)"으로 잡았다. 그러나 v1 규모(1만~10만)에선 스케일은 브루트포스로 무감각하고, 인덱스 잡음은 랭킹이 묻으며, 프라이버시는 하드 게이트가 처리한다. 즉 공격적 soft gate가 막는 것은 거의 잡음뿐인데 그 대가가 복구 불가능한 손해였다.

## Consequences

- 성공 지표를 "게이트 통과율(많이 걸러짐=좋음)"에서 "회수 성공률(질문->정답 #1)"로 바꾼다. 잡음률을 칭찬하지 않는다.
- Soft gate는 하드 컷이 아니라 OR 조건으로 둔다(예: dwell 10초 전이라도 스크롤·체류 신호가 있으면 담는다).
- Manual save는 게이트 철학과 독립적으로, 사용자가 false negative를 직접 메우는 탈출구다.
- 1-pager의 "engagement-gated capture가 가장 레버리지 큰 결정" 및 "절반↑ 걸러져야 함"은 이 ADR로 대체된다.
