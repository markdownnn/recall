# 랭킹: recency 순가점 없음, 하이라이트는 청크 가점

회수 랭킹의 1축은 의미 유사도다. recency를 "최근일수록 가점"으로 쓰지 않는다 — 이 제품은 오래된 자료를 찾으려는 것이고(최근 건 브라우저 히스토리로 찾음), 최근 가점은 오래된 정답을 아래로 밀어 핵심 가치를 거스른다. 하이라이트된 Chunk는 사용자가 콕 집은 최고신호이므로 의미 점수에 가점(보너스 곱)을 준다.

## Considered Options

일반 검색·피드의 관성으로 recency 순가점을 넣기 쉽지만, 그것은 "건초더미가 커져도 정답 1등 유지"라는 이 제품의 목표와 정반대다. recency를 굳이 쓴다면 순가점이 아니라 아주 약한 동점 처리(tie-break)까지만 허용한다.

## Consequences

- 미래에 누군가 "검색인데 최신 가점이 없네, 버그 아냐?" 하고 recency 가점을 추가하려 할 수 있다. 그건 의도된 설계이며, 바꾸려면 이 ADR을 먼저 재검토해야 한다.
- 하이라이트는 항상 자기만의 Chunk로 따로 임베딩한다(ADR 0006 참조). 큰 자동 청크에 녹이면 사용자가 콕 집은 최고신호의 의미가 평균에 희석돼 정밀도를 잃기 때문이다. 가점은 이 하이라이트 Chunk에만 주고 같은 페이지의 다른 Chunk로 번지지 않는다. 자동 청크와 텍스트가 겹치는 중복은 검색 시점 dedup으로 합친다(하이라이트 카드 우선).
