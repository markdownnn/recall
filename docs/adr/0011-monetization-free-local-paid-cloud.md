# 유료화: 무료 로컬 / 유료 클라우드층

v1은 완전한 로컬 경험을 무료로 제공하고, 유료층은 클라우드 보조 기능(기기 간 동기화/백업이 주력, 선택적 서버 임베딩이 보조)으로 둔다. 무료를 불구로 만들지 않는다. 유료/무료 경계는 지금 긋되, 유료 기능을 v1에 만들 필요는 없다 -- 단지 무료로 주지 않을 뿐이다(claw back 방지).

## Considered Options

local-first는 차별점이지만 한계비용 0이라 "내 기기에서 도는데 왜 구독?"이라는 인식 때문에 순수 로컬 기능의 구독 정당화가 어렵다. 또 순수 로컬 유료 기능은 라이선스를 걸어도 크랙되어 단속이 약하다. 반면 동기화/서버임베딩은 (1) 실제 서버 비용이 들어 구독이 정직하고 (2) 서버를 쥐고 있어 단속이 가능하다. 한 번 사는 유료 앱 모델도 있으나, "오래될수록 가치가 오른다"는 제품 명제가 시간이 갈수록 백업/동기화 지불의사를 키우므로 구독형 클라우드층이 더 자연스럽다.

## Consequences

- 유료 wedge는 `StoragePort` 뒤 sync 어댑터(현재 OUT)와 `EmbeddingPort` 뒤 ServerEmbeddingAdapter(ADR 0001 escape hatch)다. 둘 다 이미 포트로 자리만 뚫려 있다.
- 전부 무료로 뿌리면 앵커링+claw back으로 유료화가 막힌다. "클라우드층만 무료로 주지 않기"로 활주로를 연다.
- 동기화/백업은 데이터 분실 방지라는 점에서 ADR 0007/durability 논의와도 이어진다(로컬 단일 기기 보관의 분실 위험을 유료층이 해소).
