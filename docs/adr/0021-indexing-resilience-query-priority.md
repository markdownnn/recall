# 색인 회복력 + 대화형 질의 우선순위

캡처는 청크를 **즉시 저장**(vector NULL)하고 임베딩은 백그라운드에서 채운다(ADR 0017의 영속 큐). 그런데 "검색이 멈춘다 / 결과가 안 나온다"는 증상이 있었다 — 원인은 대화형 **질의(query) 임베딩이 줄 서 있던 백그라운드 색인(passage) 임베딩 뒤에 갇히는 우선순위 역전**이었다. 이를 네 갈래로 고쳤다: (a) 두 레인 single-flight 스케줄러로 질의를 앞세우고, (b) drain이 일시적 실패를 재시도하되 영구 실패엔 우아하게 멈추며, (c) keep-alive ping이 drain을 다시 깨워 누락 청크를 잇고, (d) 색인 배치를 8로 줄이고 GPU-양보 틈을 둬 보던 페이지가 매끄럽게 유지된다.

## 무엇을 고쳤나

### (a) 질의 우선 두 레인 스케줄러 — `src/offscreen/webgpu-embedder.ts`
임베더는 `highQ`(query)·`lowQ`(passage) 두 큐를 둔다. `pump`는 **항상 highQ를 먼저** 비운다. ONNX는 한 번에 입력 하나만(겹치면 안 됨) 받으므로 single-flight이고, 진행 중인 `runEmbed`는 끝까지 간다(ONNX는 중간에 못 끊는다). 그래서 대화형 검색이 ~10초씩 멈추던 stall이 사라진다 — 질의는 줄 맨 앞으로 끼어든다.

### (b) drain 재시도 + 우아한 정지 — `src/core/indexing-service.ts`
`embedBatchWithRetry`는 일시적 임베딩 실패를 지수 백오프(250/500/1000ms)로 재시도한다. `maxRetries` 후에도 실패하면 `false`를 반환하고 `drain`은 **루프를 멈춘다** — throw하지도, 무한 반복하지도 않고 나머지 청크를 그냥 pending으로 남긴다. 캡처는 fire-and-forget로 drain을 돌리므로, 여기서 throw하면 팝업이 영영 "indexing..."에 묶인다.

### (c) keep-alive ping이 drain 재가동 — `src/offscreen/offscreen.ts`
SW가 보내는 `ping` op이 `runDrainWithProgress()`를 다시 호출한다. (b)에서 영구 실패로 남겨졌거나 인터럽트로 갇힌 청크가 나중에 **반드시 따라잡는다**. drain은 single-flight + 빈 큐 즉시 반환이라, 할 일이 없으면 이 ping은 공짜다.

### (d) 배치 8 + GPU-양보 틈 — `webgpu-embedder.ts` / `indexing-service.ts`
색인 배치를 8로 줄였다. 큰 단일 제출은 GPU를 독점해 **사용자가 지금 읽는 페이지를 버벅이게** 한다. 작은 제출 사이에 120ms(`YIELD_MS`)를 양보하면 포그라운드가 계속 렌더되고, 그 틈에 끼어든 질의도 시작할 수 있다. 질의는 보통 1텍스트=1배치라 배치 간 yield를 아예 안 친다 → 검색은 빠르다.

## Considered Options / 트레이드오프

- **ONNX는 인터럽트 불가** → 질의는 **진행 중인 작은 배치 하나**를 끝까지 기다린다(0은 못 만든다). 그래서 배치를 8로 작게 둔다 — 배치가 작을수록 질의가 무는 최악 대기가 짧다. "질의를 칼같이 즉시"가 아니라 "최대 작은 배치 하나만큼"이 의도된 상한이다.
- **이론상 질의가 passage를 굶길 수 있다.** highQ를 항상 먼저 비우니, 질의가 끝없이 쏟아지면 색인이 영영 안 된다. 하지만 질의는 사람이 타이핑하는 속도(human-paced)라 현실적으로 색인이 굶지 않는다. 굶더라도 ping(c)이 빈 틈에 drain을 잇는다.
- **남겨둔 pending은 손실이 아니다.** vector NULL 청크가 곧 영속 큐다(ADR 0017). 재시작·재드레인에서 그대로 이어진다 — 그래서 영구 실패에 throw 대신 "그냥 둔다"가 안전하다.

## Consequences

- 대화형 검색이 백그라운드 색인 중에도 막히지 않는다(우선순위 역전 해소).
- 일시적 실패는 자가 치유(재시도), 영구 실패는 제품을 안 죽이고 나중에 따라잡힘(ping). 팝업이 "indexing..."에 영영 묶이는 경로가 사라진다.
- 포그라운드 페이지가 색인 중에도 매끄럽다(배치 8 + 120ms yield).
- 미래에 "왜 임베딩 배치가 이렇게 작지?" 하면 이 ADR을 본다 — 작은 배치는 질의 대기 상한을 짧게 만들고 GPU 독점을 막으려는 의도다.
